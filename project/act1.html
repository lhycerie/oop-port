Lab Activity 1

Variables & Memory Analysis

Midterms Java OOP

Explanation & Learnings

In this activity, I analyzed a Java program to trace object references and memory changes step-by-step. The main focus was understanding how variables are stored in memory—specifically the difference between the Stack (for local variables and references) and the Heap (for instance variables and actual object data).

  • Memory Management: A Student object is created in the Heap, while the reference variables (`s1`, `s2`, `s3`) are stored in the Stack holding the memory addresses.
  • Shared References: I learned that when multiple variables point to the same object (like s3 = s2), they share the exact same Heap data.
  • Data Mutation: Because s3 and s2 pointed to the same object, changing s3.name to "Ana" also changed what s2 outputted, permanently overwriting "Maria" in the Heap.

The Code Execution

Below is the logic analyzed during the activity, focusing on tracking memory states across object instantiation and mutation.

TestStudent.java 
class Student {
    String name;
    int grade;

    Student(String n, int g) {
        name = n;
        grade = g;
    }
}

public class TestStudent {
    public static void main(String[] args) {
        Student s1 = new Student("John", 85);
        Student s2 = new Student("Maria", 90);
        Student s3 = s2;
        
        s1.grade = 95;
        s3.name = "Ana";
        
        System.out.println(s1.name + " " + s1.grade);
        System.out.println(s2.name + " " + s2.grade);
        System.out.println(s3.name + " " + s3.grade);
    }
}
